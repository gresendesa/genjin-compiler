## Decisões técnicas

### Fixture `run_genjin`
- Lê o template como texto e o passa via **stdin** ao `assembler.py - -d code/`
- Evita o problema do DottedLoader (que converte pontos em separadores de diretório)
- `expect_error=False` → assert returncode 0, retorna stdout
- `expect_error=True` → assert returncode != 0, retorna stderr

### Normalização de IDs nas comparações
- `Cortex` gera IDs hexadecimais aleatórios (ex.: `result7a84f1`)
- Função `normalize_ids(text)` substitui sufixo hex de 6 chars por `XXXXXX`
- Golden files usam `XXXXXX` no lugar dos IDs

### Acesso a macros internas do genjin.jinja2
- `from "genjin" import get_unhandled_codes_recursively` é válido — macros Jinja2 não têm modificador de acesso
- Macros importadas retêm referência às próprias globais (ex.: `get_unhandled_codes` permanece acessível internamente)

### CORE mock nos templates unitários
- Dicionário Jinja2 com macros inline como `procedure.find.by_name`
- Suficiente para os testes de U-10 e U-11 que não exercitam CALL_PROCEDURE
