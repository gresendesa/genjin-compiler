## Decisões de design

### Serialização da AST (S3-T01)

- Estratégia: `dataclasses.asdict()` + campo `__type__` com nome da classe para reconstrução
- Módulo: `compiler/ast_io.py` com funções `ast_to_json(node) -> str` e `ast_from_json(s) -> ProgramNode`
- Tipos especiais: `bool` preservado como bool JSON; `int` como number; `None` como null
- Nós recursivos (`ExecBlockNode` → `CaseNode` → `ExecBlockNode`) tratados pelo campo `__type__`

### Convenção de stdin/arquivo

- Argumento posicional opcional: se ausente, lê stdin
- `--source` nas etapas tardias (parser, transpiler) para rodar o pipeline desde o início

### Tratamento de erros

- `ScannerError` e `ParseError`: mensagem para stderr, exit code 1
- Erro de leitura de arquivo: mensagem para stderr, exit code 2

---

## Bloqueios previstos e mitigações

| Bloqueio | Mitigação |
|---|---|
| Desserialização de AST com nós aninhados e tipos enum | Usar campo `__type__` e função de despacho por tipo de nó |
| Testes de CLI com subprocess podem ser frágeis em CI | Usar `subprocess.run` com `capture_output=True` e fixture de arquivo temporário |
