## Decisões técnicas

### Abordagem: `OBJECT_LITERAL` no scanner via paren-depth

O scanner rastreia `_paren_depth` (incrementado em `(`, decrementado em `)`).

| Char | Condição | Token emitido |
|---|---|---|
| `[` | próximo char é `]` | `LBRACKET` (plural `Text[]`) |
| `[` | `_paren_depth > 0` e próximo char ≠ `]` | `OBJECT_LITERAL` (captura balanceada) |
| `[` | `_paren_depth == 0` | `LBRACKET` (fallback) |
| `{` | `_paren_depth > 0` | `OBJECT_LITERAL` (captura balanceada) |
| `{` | `_paren_depth == 0` | `LBRACE` (bloco) |

O método `_read_collection_literal(open_char, close_char)` captura o conteúdo bruto com:
- rastreamento de profundidade para aninhamento
- respeito a aspas simples e duplas (não conta delimitadores dentro de strings)
- atualização de `self._line` em newlines
- retorna a string completa incluindo delimitadores de abertura e fechamento

### No parser (`_parse_kwargs`)

Adicionar ramo `OBJECT_LITERAL` após o ramo `STRING`:
- Se `param.type != 'object'` → `ParseError` descritivo
- Caso contrário → `ArgNode(value=token.value, evaluation='literal', raw=True)`
