**Saída:** AST (Abstract Syntax Tree)

## Estrutura da AST

A AST deve espelhar fielmente os conceitos do motor Genjin (`docs/genjin.md`), mapeados pela sintaxe `.gnj` (`docs/language.md`).

### Nós principais

```python
@dataclass
class ProgramNode:
    name: str
    variables: list[VarDeclNode]
    procedures: list[ProcDeclNode]
    block: ExecBlockNode

@dataclass
class VarDeclNode:
    name: str
    type: str           # 'number' | 'text' | 'logic'
    cardinality: str    # 'singular' | 'plural'
    value: Any          # None se não declarado

@dataclass
class ProcDeclNode:
    name: str
    library: str        # parte antes do último ponto do `from`
    macro: str          # parte após o último ponto do `from`
    parameters: list[ParamDeclNode]
    output_codes: list[OutputCodeNode]

@dataclass
class ParamDeclNode:
    name: str
    type: str
    cardinality: str    # 'singular' por padrão
    evaluation: str     # 'literal' | 'reference'

@dataclass
class OutputCodeNode:
    name: str
    code: int

@dataclass
class ExecBlockNode:
    proc_name: str
    kwargs: dict[str, ArgNode]
    variable: str | None    # None = herda do pai
    block_name: str | None  # None = usa nome do proc
    cases: list[CaseNode]
    loop_while: list[str]
    pass_codes: list[str]

@dataclass
class CaseNode:
    output_code: str
    block: ExecBlockNode

@dataclass
class ArgNode:
    value: Any
    evaluation: str     # 'literal' | 'reference'
```

## Regras semânticas a validar no parser

1. Todo código declarado em `case` ou `while` deve corresponder a um código declarado no `proc` referenciado.
2. Todo `pass` deve listar códigos não tratados por `case` ou `while`.
3. Toda variável referenciada em `>>` ou `=&` deve estar declarada em `vars`.
4. Todo `proc` referenciado em `exec` deve estar declarado em `procs`.
5. Parâmetro com `&` na declaração só aceita argumento `=&var` (referência), nunca literal.
