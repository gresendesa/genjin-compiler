## Contexto

A linguagem Genjin (`.gnj` / `.genjin`) não possui suporte nativo no VS Code.
O objetivo é criar uma extensão que registre a linguagem e forneça syntax highlighting
básico, preparando a estrutura para evoluir para autocomplete, diagnostics e Language Server.

## Escopo

Criar o projeto mínimo da extensão no diretório `vscode-genjin/` dentro do repositório principal:

- `vscode-genjin/package.json` — manifesto da extensão (contributes: languages, grammars)
- `vscode-genjin/language-configuration.json` — comentários, brackets, auto-closing pairs
- `vscode-genjin/syntaxes/genjin.tmLanguage.json` — TextMate Grammar básico
- `vscode-genjin/README.md` — documentação da extensão

### Funcionalidades do escopo inicial

1. Registrar linguagem `genjin` no VS Code
2. Associar arquivos `.gnj` e `.genjin` à linguagem
3. Syntax highlighting via TextMate Grammar cobrindo:
   - Keywords: `program`, `vars`, `procs`, `proc`, `from`, `exec`, `case`, `pass`, `while`, `as`, `codes`
   - Tipos: `Number`, `Text`, `Logic`
   - Literais: strings, números inteiros
   - Operadores: `>>`, `&`, `=`, `<`, `>`
   - Comentários de linha (`//`)
4. Language configuration com:
   - Comentário de linha: `//`
   - Comentário de bloco: `/* */`
   - Brackets: `{}`, `()`, `[]`, `<>`
   - Auto-closing pairs para strings, brackets e comentários

### Fora do escopo inicial

- Language Server Protocol (LSP)
- Autocomplete / IntelliSense
- Diagnostics em tempo real
- Formatador automático

## Estrutura de evolução futura

A extensão será organizada para suportar adição futura de:
- `server/` — Language Server (Node.js ou Python via `pygls`)
- `client/` — cliente LSP integrado ao VS Code
