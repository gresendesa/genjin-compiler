# B-013 — Parser: remover keyword `proc` das declarações dentro de `procs { }`

- Status: pendente
- Prioridade PO: 1 (crítica)
- Owner: agente
- Data de criacao: 2026-05-01
- Sprint: SPR-2026-06

## Contexto

Atualmente, cada procedimento dentro do bloco `procs { }` exige a keyword `proc` antes do nome:

```gnj
procs {
    proc inicia_processo() from "tests.default.xpto" {
        codes OK<0>, ERROR<1>
    }
}
```

A sintaxe desejada é que a keyword `proc` seja omitida — o contexto do bloco `procs { }` já
indica que o que está dentro são declarações de procedimento:

```gnj
procs {
    inicia_processo() from "tests.default.xpto" {
        codes OK<0>, ERROR<1>
    }
}
```

## Escopo

1. **`compiler/scanner.py`** — remover o token `KW_PROC` (ou mantê-lo apenas para evitar colisão, marcando como reservado sem uso ativo).
2. **`compiler/parser.py`** — método `_parse_proc_decl`: remover `_expect(TokenType.KW_PROC, ...)`.
3. **`examples/basic.gnj` e `examples/test.gnj`** — remover `proc` das declarações.
4. **`tests/`** — atualizar fixtures/templates que usem a sintaxe antiga.
5. **`docs/language.md`** — atualizar exemplos de declaração de proc.

### Gramática esperada após a correção

```
proc_decl := IDENT '(' param_list ')' 'from' STRING '{' 'codes' code_list '}'
```

## Critérios de aceite

- [ ] `procs { nome_proc(...) from "..." { codes ... } }` é aceito pelo parser
- [ ] `procs { proc nome_proc(...) from "..." { ... } }` lança `ParseError` com mensagem clara
- [ ] Todos os testes existentes passam (após atualização das fixtures)
- [ ] `examples/basic.gnj` e `examples/test.gnj` atualizados
- [ ] `docs/language.md` atualizado

## Dependências

Nenhuma.

## Risco

Baixo — mudança de remoção simples no parser; nenhum impacto em AST, transpiler ou assembler.
