## Escopo

Criar `compiler.py` na raiz do projeto (mesmo diretório de `assembler.py`) que executa o pipeline completo: scanner → parser → transpiler.

Este script é o ponto de entrada principal para o usuário que quer compilar um `.gnj` direto para um template Jinja2 sem se preocupar com etapas individuais.

## Comportamento esperado

```bash
# Lê arquivo, imprime template para stdout
python compiler.py programa.gnj

# Lê stdin, imprime template para stdout
cat programa.gnj | python compiler.py

# Escreve para arquivo
python compiler.py programa.gnj -o saida.jinja2
python compiler.py programa.gnj > saida.jinja2
```

## Interface CLI

```
uso: compiler.py [arquivo] [-o saida]

argumentos posicionais:
  arquivo          Arquivo .gnj de entrada (padrão: stdin)

opções:
  -o, --output     Arquivo de saída (padrão: stdout)
  -h, --help       Exibe ajuda
```

## Critérios de aceitação

- [ ] `python compiler.py examples/basic.gnj` imprime template Jinja2 válido para stdout.
- [ ] `cat examples/basic.gnj | python compiler.py` funciona via stdin.
- [ ] `-o arquivo` escreve saída para arquivo em vez de stdout.
- [ ] Erros (ScannerError, ParseError) saem para stderr com exit code != 0 e mensagem legível.
