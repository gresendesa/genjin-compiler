## Contexto

B-019 estende o parser para emitir `ProcBlockNode` na AST. B-020 cobre a segunda fase:
ensinar o desugar (`compiler/desugar.py`) a **expandir** as chamadas de proc-bloco — substituindo
`@ProcBloco(params)` pela AST clonada e parametrizada do corpo — produzindo uma `ProgramNode`
canônica que o transpiler já consegue processar sem alterações.

## Escopo

### 1. Detecção de recursão (DFS)

No início de `desugar()`, após construir o `proc_map`, construir `block_map: dict[str, ProcBlockNode]`
com apenas os proc-blocos. Fazer DFS sobre o grafo de dependências entre proc-blocos para
detectar ciclos.

**Grafo:** `A → B` se o corpo de `A` contém uma chamada `@B` ou `exec B()` onde `B` é
`ProcBlockNode`.

**Algoritmo:** coloração padrão (branco/cinza/preto). Ciclo detectado → `DesugarError` com
o caminho do ciclo.

```python
class DesugarError(Exception):
    ...

# No início de desugar():
_detect_proc_block_cycles(block_map)
```

Estimativa: ~30 linhas.

### 2. `proc_map` passa a incluir `ProcBlockNode`

```python
# antes
proc_map: dict[str, ProcDeclNode]
# depois
proc_map: dict[str, ProcDeclNode | ProcBlockNode]
```

As funções existentes `_build_terminal` e `_build_chained` que consultam `proc_map` precisam
distinguir os dois tipos. Quando o alvo é `ProcBlockNode`, delegar para o novo passo de expansão.

### 3. Ordem topológica de expansão

A DFS do passo 1 produz ordem topológica (folhas primeiro — proc-blocos sem dependências em
outros proc-blocos). A expansão deve seguir essa ordem: expandir `A_Bloco` antes de `B_Bloco`
se `B_Bloco` usa `A_Bloco`. Isso garante que ao expandir `B_Bloco`, o corpo de `A_Bloco` já
está completamente expandido (sem mais referências a outros proc-blocos).

### 4. Expansão de `@ProcBloco(params)` e `exec ProcBloco(params)`

Quando `_build_terminal` (ou `_build_chained`) encontra um alvo `ProcBlockNode`:

```python
def _expand_proc_block(
    proc_block: ProcBlockNode,
    call_kwargs: dict[str, ArgNode],
) -> ExecBlockNode:
    # 1. Validar que todos os parâmetros têm argumento
    # 2. Construir mapeamento placeholder → ArgNode
    substitution = {p.name: call_kwargs[p.name] for p in proc_block.parameters}
    # 3. Deep clone do corpo
    cloned = copy.deepcopy(proc_block.block)
    # 4. Aplicar visitor de substituição
    _substitute_params(cloned, substitution)
    # 5. Retornar o clone substituído (já é ExecBlockNode canônico)
    return cloned
```

Estimativa: ~25 linhas (sem contar o visitor).

### 5. Visitor de substituição de parâmetros

Função recursiva que percorre a AST substituindo `ArgNode` cujo `value` bate com uma chave de
`substitution`:

```python
def _substitute_params(node, substitution: dict[str, ArgNode]) -> None:
    if isinstance(node, ExecBlockNode):
        for key, arg in node.kwargs.items():
            if arg.value in substitution:
                node.kwargs[key] = copy.deepcopy(substitution[arg.value])
        for case in node.cases:
            _substitute_params(case.block, substitution)
    elif isinstance(node, CaseNode):
        _substitute_params(node.block, substitution)
    elif isinstance(node, InlineSeqNode):
        ...  # tratar inline atoms se necessário
```

Estimativa: ~25 linhas.

### 6. Filtro de proc-blocos em `procedures`

Após a expansão, o `ProgramNode` retornado pelo desugar deve ter `procedures` contendo apenas
`ProcDeclNode` (proc-blocos removidos). O transpiler itera `procedures` esperando `ProcDeclNode`.

```python
# No final de desugar(), antes do return:
filtered_procs = [p for p in ast.procedures if isinstance(p, ProcDeclNode)]
return ProgramNode(
    name=ast.name,
    variables=ast.variables,
    procedures=filtered_procs,
    block=desugared_block,
)
```

Estimativa: ~5 linhas (já dentro do fluxo existente).

        super().__init__(message)

## Casos de teste (mínimos)

1. **Expansão simples:** proc-bloco com 1 param lit — resultado equivale ao bloco com valor inline
2. **Expansão com ref:** proc-bloco com param `ref` — variável real substituída corretamente
3. **Duas chamadas distintas:** dois `@ProcBloco` com params diferentes → dois blocos independentes
4. **Recursão direta:** `A` chama `@A` → `DesugarError` com mensagem de ciclo
5. **Recursão indireta:** `A → B → A` → `DesugarError` com caminho completo
6. **Composição sem ciclo:** `B` usa `A_Bloco` — expandido corretamente sem erro
7. **Filtro de procedures:** `ProcBlockNode` não aparece em `procedures` do Jinja2 gerado
8. **Proc-bloco vazio de params:** nenhum parâmetro → expansão de bloco fixo

## Risco

Médio. Pontos críticos:
1. **Visitor de substituição:** precisa cobrir todos os nós relevantes sem deixar placeholders
   não substituídos silenciosamente. Mitigado por testes com parâmetros em posições distintas
   (kwargs de exec raiz, de exec aninhado, de exec dentro de case).
2. **Deep clone:** `copy.deepcopy` de nós AST (`dataclass`) é seguro em Python padrão; sem risco
   de aliasing.
3. **Ordem topológica:** se a DFS estiver correta, a ordem de expansão é garantida. Testar
   explicitamente com composição de dois níveis (A usa B_Bloco, B_Bloco usa C_Bloco).
