# B-010 — Transpiler: ajustar import de genjin.jinja2 para notação DottedLoader

- Status: concluído (SPR-2026-05)
- Prioridade PO: 1 (critica)
- Owner: agente
- Data de criacao: 2026-04-28
- Data de conclusao: 2026-04-28

## Contexto

O transpiler gera a primeira linha do template assim:

```jinja2
{* from "code/genjin.jinja2" import ATTRIBUTE, TYPE, CARDINALITY, EVALUATION, MACROMOD, build *}
```

O assembler, porém, usa `DottedLoader`, que:
1. Converte pontos em separadores de diretório (`genjin` → `genjin.jinja2`).
2. Assume a extensão `.jinja2` implicitamente.
3. Resolve caminhos relativos ao diretório informado em `-d`.

Quando o assembler usa `code/` como diretório de templates (`-d ./code`), o caminho correto para referenciar `code/genjin.jinja2` é simplesmente `"genjin"`.

O caminho `"code/genjin.jinja2"` com barras e extensão explícita **não é válido** no contexto do `DottedLoader`.

## Escopo

Alterar o transpiler para emitir:

```jinja2
{* from "genjin" import ATTRIBUTE, TYPE, CARDINALITY, EVALUATION, MACROMOD, build *}
```

em vez de:

```jinja2
{* from "code/genjin.jinja2" import ATTRIBUTE, TYPE, CARDINALITY, EVALUATION, MACROMOD, build *}
```

A mudança é apenas na string emitida pelo transpiler — sem alteração na lógica de compilação.

## Impacto

- `compiler/transpiler.py`: alterar a constante/string da linha de import.
- `tests/transpiler/test_transpiler.py`: atualizar os testes que verificam a linha de import.
- Nenhuma mudança em `assembler.py`, `compiler.py`, `compiler/scanner.py` ou `compiler/parser.py`.
- `code/genjin.jinja2` permanece INTOCÁVEL (Rule 10).

## Critérios de aceite

- [ ] Primeira linha gerada pelo transpiler é `{* from "genjin" import ATTRIBUTE, TYPE, CARDINALITY, EVALUATION, MACROMOD, build *}`.
- [ ] Pipeline `python compiler.py programa.gnj | python assembler.py - -d ./code` renderiza sem `TemplateNotFound` para o genjin (depende de outras libraries do programa também estarem em `code/`).
- [ ] Todos os testes de transpiler passam com o novo formato.
- [ ] `python -m pytest tests/ -q` — 0 falhas.

## Dependências

- Nenhuma.

## Risco

- Baixo — mudança de string única com impacto localizado no transpiler e seus testes.
