# B-016 — Extensão VS Code: suporte à sintaxe inline `@proc()` e keyword `when`

- Status: concluído
- Prioridade PO: 1 (crítica)
- Owner: agente
- Data de criacao: 2026-05-02
- Sprint: SPR-2026-09

## Contexto

A SPR-2026-08 implementou a notação inline com dois novos elementos sintáticos:

1. **Keyword `when`** — usada em sequências inline do tipo `@proc() when(CODE)`
2. **Notação `@proc()`** — chamada de procedure prefixada com `@`, com destaque como entity/função

A extensão `vscode-genjin` (arquivo `vscode-genjin/syntaxes/genjin.tmLanguage.json`) não reflete
essas adições: `when` não está na lista de keywords e `@proc()` não tem nenhuma regra de highlight.

Sem essa atualização, arquivos `.gnj` que usam a sintaxe da SPR-2026-08 ficam sem colorização
adequada no VS Code.

---

## Escopo

### 1. Adicionar `when` às keywords de controle

No grupo `keywords → keyword.control.genjin`, adicionar `when` junto a `vars`, `procs`, `exec`,
`case`, `pass`, `while`.

### 2. Adicionar regra de highlight para notação inline `@proc()`

Criar uma nova regra no repositório (ex.: `inline-call`) que reconheça o padrão:

```
@identificador(...)
```

Sugestão de captura:
- `@` → `keyword.operator.inline.genjin`
- nome do proc → `entity.name.function.genjin`
- `(` e `)` → `punctuation.definition.parameters.genjin`

### 3. Adicionar `when` ao snippet/exemplo de colorização (se existir)

Verificar se há arquivo de exemplo de linguagem ou testes de tmLanguage e atualizar para cobrir
os novos padrões.

---

## Critérios de aceite

- [ ] Arquivo `examples/test.gnj` abre no VS Code com colorização correta para `when`, `@proc()` e `while` inline
- [ ] `when` aparece como keyword de controle (mesma cor de `case`, `pass`)
- [ ] `@inicia_processo()` tem o `@` e o nome da função destacados
- [ ] Nenhuma regressão nos padrões existentes (program, vars, procs, exec, codes, tipos, strings, números)
- [ ] `package.json` da extensão não requer bump de versão obrigatório, mas recomenda-se incrementar patch
