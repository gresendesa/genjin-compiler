## Escopo

Implementar a Etapa 1 do compilador: análise lexográfica de um arquivo `.gnj`.

**Entrada:** string com código-fonte `.gnj`
**Saída:** lista de objetos `Token`

## Tokens a reconhecer

### Keywords

| Token | Lexema |
|---|---|
| `KW_PROGRAM` | `program` |
| `KW_VARS` | `vars` |
| `KW_PROCS` | `procs` |
| `KW_PROC` | `proc` |
| `KW_FROM` | `from` |
| `KW_EXEC` | `exec` |
| `KW_CASE` | `case` |
| `KW_PASS` | `pass` |
| `KW_WHILE` | `while` |
| `KW_AS` | `as` |

### Tipos primitivos

| Token | Lexema |
|---|---|
| `TYPE_NUMBER` | `Number` |
| `TYPE_TEXT` | `Text` |
| `TYPE_LOGIC` | `Logic` |

### Símbolos

| Token | Lexema |
|---|---|
| `LBRACE` | `{` |
| `RBRACE` | `}` |
| `LPAREN` | `(` |
| `RPAREN` | `)` |
| `LANGLE` | `<` |
| `RANGLE` | `>` |
| `LBRACKET` | `[` |
| `RBRACKET` | `]` |
| `COLON` | `:` |
| `COMMA` | `,` |
| `AMPERSAND` | `&` |
| `ASSIGN` | `=` |
| `ARROW` | `>>` |

### Literais e identificadores

| Token | Descrição |
|---|---|
| `STRING` | String entre aspas duplas: `"valor"` |
| `NUMBER` | Número inteiro: `0`, `5`, `10` |
| `IDENT` | Identificador: nome de variável, proc, código de saída |

### Especiais

| Token | Descrição |
|---|---|
| `EOF` | Fim do arquivo |

Comentários (`//` e `/* */`) devem ser ignorados (não geram tokens).

## Estrutura de dados esperada

```python
from dataclasses import dataclass
from enum import Enum, auto

class TokenType(Enum):
    # keywords, tipos, símbolos, literais, EOF...

@dataclass
class Token:
    type: TokenType
    value: str
    line: int
```

## Critérios de aceitação

- [ ] Todos os token types listados reconhecidos corretamente.
- [ ] Comentários `//` e `/* */` ignorados.
- [ ] Strings, números e identificadores capturados com valor correto.
- [ ] `ARROW` (`>>`) reconhecido antes de `RANGLE` (`>`).
- [ ] Número de linha rastreado em cada token.
- [ ] Erro claro para caracteres não reconhecidos.
- [ ] Testes em `tests/` cobrindo o scanner.
