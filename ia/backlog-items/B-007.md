## Escopo

Tornar cada script de etapa (`compiler/scanner.py`, `compiler/parser.py`, `compiler/transpiler.py`) executável diretamente pela linha de comando, com entrada por stdin ou arquivo e saída para stdout.

Isso permitirá compor o pipeline via shell pipes e, no futuro, fazer cache seletivo de etapas custosas.

## Comportamento esperado por etapa

### scanner.py

```bash
python -m compiler.scanner < entrada.gnj        # lê stdin
python -m compiler.scanner entrada.gnj          # lê arquivo
```

**Saída (stdout):** JSON com lista de tokens.
```json
[
  {"type": "KW_PROGRAM", "value": "program", "line": 1},
  ...
]
```

### parser.py

```bash
python -m compiler.parser < tokens.json
python -m compiler.parser tokens.json
python -m compiler.parser --source entrada.gnj   # atalho: tokeniza e parseia
```

**Entrada:** JSON de tokens (saída do scanner) OU `--source` para rodar scanner internamente.
**Saída (stdout):** JSON com a AST serializada.

### transpiler.py

```bash
python -m compiler.transpiler < ast.json
python -m compiler.transpiler ast.json
python -m compiler.transpiler --source entrada.gnj   # atalho: pipeline completo
```

**Entrada:** JSON da AST (saída do parser) OU `--source` para rodar pipeline internamente.
**Saída (stdout):** Template Jinja2 (string).

## Pipeline via shell

```bash
# Pipeline completo
cat programa.gnj | python -m compiler.scanner | python -m compiler.parser | python -m compiler.transpiler

# Com cache da etapa scanner (por exemplo)
python -m compiler.scanner programa.gnj > tokens.json
python -m compiler.parser tokens.json | python -m compiler.transpiler
```

## Serialização da AST

A AST precisa de uma estratégia de serialização/desserialização para circular entre etapas via JSON. Cada nó deve ser serializável com `dataclasses.asdict()` + um campo `__type__` para reconstrução.

## Critérios de aceitação

- [ ] `python -m compiler.scanner arquivo.gnj` imprime JSON de tokens para stdout.
- [ ] `python -m compiler.scanner < arquivo.gnj` funciona igualmente via stdin.
- [ ] `python -m compiler.parser < tokens.json` imprime JSON da AST.
- [ ] `python -m compiler.parser --source arquivo.gnj` também funciona.
- [ ] `python -m compiler.transpiler < ast.json` imprime template Jinja2.
- [ ] `python -m compiler.transpiler --source arquivo.gnj` também funciona.
- [ ] Pipeline `scanner | parser | transpiler` via shell funciona end-to-end.
- [ ] Erros (ScannerError, ParseError) saem para stderr com exit code != 0.
- [ ] Testes em `tests/` cobrindo os modos CLI de cada etapa.
