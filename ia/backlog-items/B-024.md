## Contexto

A notação inline (`@proc() while(CODE) when(CODE)`) possui uma restrição injustificada em
`compiler/parser.py` (`_parse_inline_atom`):

1. `while` inline aceita **apenas 1 código** (sem vírgula entre múltiplos)
2. O código é **validado contra `valid_codes` do próprio proc** — rejeita códigos borbulhados
   de procs filhos

O `while` canônico (`_parse_while`, usado por `exec {}`) já funciona corretamente: aceita
múltiplos códigos com vírgula e **não valida** contra o proc (permite borbulhamento).

A restrição no inline é inconsistente com o canônico e bloqueia a refatoração de
`Troca_Ferramenta` (B-023): `Definir_Home` teria `while(DEMORA, CANCELADO)` onde `CANCELADO`
é borbulhado de `Teleportar` filho.

---

## Escopo

### 1. Corrigir `_parse_inline_atom` em `compiler/parser.py`

Substituir o parsing de `while` inline de código único + validação por loop com vírgula
sem validação contra `valid_codes`, alinhando com `_parse_while`:

```python
# Antes (linha ~406):
if self._check(TokenType.KW_WHILE):
    self._advance()
    self._expect(TokenType.LPAREN, ...)
    code_tok = self._expect(TokenType.IDENT, ...)
    if code_tok.value not in valid_codes:          # ← restrição injustificada
        raise ParseError(...)
    self._expect(TokenType.RPAREN, ...)
    while_code = code_tok.value                    # ← apenas 1 código

# Depois:
if self._check(TokenType.KW_WHILE):
    self._advance()
    self._expect(TokenType.LPAREN, ...)
    while_codes: list[str] = []
    while not self._check(TokenType.RPAREN, TokenType.EOF):
        if while_codes:
            self._expect(TokenType.COMMA, ...)
        while_codes.append(self._expect(TokenType.IDENT, ...).value)
        # SEM validação contra valid_codes — borbulhamento é válido
    self._expect(TokenType.RPAREN, ...)
```

### 2. Atualizar `InlineAtomNode`

`while_code: str | None` → `while_codes: list[str]` (ou manter `while_code` como
`list[str]` — verificar contrato atual e todos os consumidores).
