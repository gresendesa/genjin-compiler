## Escopo

Implementar a Etapa 3 do compilador: geração de template Jinja2 a partir da AST.

**Entrada:** AST (`ProgramNode` — saída do parser)
**Saída:** string com template Jinja2 pronto para execução pelo `assembler.py`

## Contrato de saída

O template gerado deve:

1. Importar as macros de procedimento via `{* from "library" import macro *}` para cada proc.
2. Importar os atributos do motor Genjin de `code/genjin.jinja2`.
3. Declarar o objeto `program` usando `ATTRIBUTE`, `TYPE`, `CARDINALITY`, `EVALUATION`.
4. Chamar `build(prog=program, language_renderer=MACROMOD)`.

### Exemplo de saída esperada para `examples/basic.gnj`

```jinja2
{* from "code/genjin.jinja2" import ATTRIBUTE, TYPE, CARDINALITY, EVALUATION, MACROMOD, build *}
{* from "Net" import check as verificar_rede *}
{* from "Sys" import sleep as esperar *}
{* from "Sys" import send as enviar *}

{*
    set program = {
        ATTRIBUTE.NAME: 'Sistema de Pagamento',
        ATTRIBUTE.VARIABLES: [ ... ],
        ATTRIBUTE.PROCEDURES: [ ... ],
        ATTRIBUTE.BLOCK: { ... }
    }
*}

{{ build(prog=program, language_renderer=MACROMOD) }}
```

## Critérios de aceitação

- [ ] Template gerado é válido Jinja2 executável pelo `assembler.py`.
- [ ] Todos os atributos da AST mapeados corretamente para `ATTRIBUTE.*`, `TYPE.*`, `CARDINALITY.*`, `EVALUATION.*`.
- [ ] Herança de variável entre blocos preservada (ausência de `ATTRIBUTE.VARIABLE` nos blocos filhos sem `>>`).
- [ ] `ATTRIBUTE.MACRO` gerado como tupla `('library', 'macro')` conforme resolução do `from`.
- [ ] Testes em `tests/` cobrindo o transpiler com `examples/basic.gnj` como fixture.
