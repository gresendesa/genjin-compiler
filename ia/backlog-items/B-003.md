## Escopo

Criar o documento `docs/language.md` com a especificação completa da sintaxe da linguagem `.gnj`, baseada no exemplo inicial em `examples/basic.gnj`.

A documentação deve cobrir:

1. Estrutura geral de um programa `.gnj`.
2. Declaração de programa (`program`).
3. Bloco de variáveis (`vars`): tipos, cardinalidade, valor inicial.
4. Bloco de procedimentos (`procs`): parâmetros, referências (`&`), origem (`from`), códigos de saída.
5. Bloco de execução (`exec`): vinculação a variável (`>>`), casos (`case`), delegação (`pass`), repetição (`while`).
6. Tipos primitivos: `Number`, `Text`, `Logic`.
7. Cardinalidade: singular vs. lista (`[]`).
8. Referência a variável por ponteiro (`&`).

## Referência principal

```gnj
program "Sistema de Pagamento"

vars {
    status_var: Number
    status_var2: Number
    status_conexao: Text = "idle"
    res: Text
    minha_lista: Text[]
}

procs {
    proc verificar_rede() from "Net.check" {
        codes ONLINE<0>, OFFLINE<1>
    }
    
    proc esperar(segundos: Number) from "Sys.sleep" {
        codes DONE<0>, ERROR<5>
    }

    proc enviar(texto: Text, resposta: &Text) from "Sys.send" {
        codes OK<0>, TIMEOUT<10>
    }
}

exec verificar_rede() >> status_var {
    case OFFLINE : exec esperar(segundos=5) {
        case DONE : exec enviar(texto="OK", resposta=&res) >> status_var2 {
            pass OK, TIMEOUT
        }
    } while(ERROR)
    
    pass ONLINE, OK, TIMEOUT
}
```

## Referências obrigatórias

A documentação da linguagem deve ser elaborada tendo como base:

- `examples/basic.gnj` — exemplo inicial da sintaxe `.gnj`. Ponto de partida.
- `docs/genjin.md` — **referência conceitual**. Define a semântica completa do motor Genjin: célula de operação, VARIABLE, PROCEDURE, OUTPUT_CODE, BLOCK, CASES, LOOP_WHILE, PASS_CODES, cadeia de responsabilidade, herança de variáveis. A linguagem `.gnj` deve mapear todos esses conceitos.
- `code/genjin.jinja2` — **referência de implementação** (INTOCÁVEL). Define os atributos concretos usados pelo motor: `ATTRIBUTE`, `TYPE`, `CARDINALITY`, `EVALUATION`, `STAGE`, `MACROMOD`. A documentação deve deixar claro como cada construto `.gnj` corresponde a esses atributos.

## Mapeamento de referência esperado

A documentação deve incluir o mapeamento explícito entre a sintaxe `.gnj` e o motor:

| Sintaxe `.gnj` | Conceito em `genjin.md` | Atributo em `genjin.jinja2` |
|---|---|---|
| `program "nome"` | Nome do programa | `ATTRIBUTE.NAME` |
| `vars { ... }` | Lista de variáveis | `ATTRIBUTE.VARIABLES` |
| `nome: Tipo` | Declaração de variável singular | `ATTRIBUTE.NAME`, `ATTRIBUTE.TYPE`, `CARDINALITY.SINGULAR` |
| `nome: Tipo = valor` | Variável com valor inicial | `ATTRIBUTE.VALUE` |
| `nome: Tipo[]` | Variável plural | `ATTRIBUTE.CARDINALITY` → `CARDINALITY.PLURAL` |
| `procs { ... }` | Lista de procedimentos | `ATTRIBUTE.PROCEDURES` |
| `proc nome(...) from "lib"` | Declaração de procedimento | `ATTRIBUTE.NAME`, `ATTRIBUTE.MACRO` |
| `param: Tipo` | Parâmetro passado por valor | `ATTRIBUTE.EVALUATION` → `EVALUATION.LITERAL` |
| `param: &Tipo` | Parâmetro passado por referência | `ATTRIBUTE.EVALUATION` → `EVALUATION.REFERENCE` |
| `codes NOME<código>` | Código de saída | `ATTRIBUTE.OUTPUT_CODES`, `ATTRIBUTE.CODE` |
| `exec proc() >> var { }` | Bloco com variável vinculada | `ATTRIBUTE.BLOCK`, `ATTRIBUTE.VARIABLE`, `ATTRIBUTE.PROCEDURE` |
| `case NOME : exec ...` | Ramificação por código | `ATTRIBUTE.CASES`, `ATTRIBUTE.OUTPUT_CODE` |
| `pass CODIGO, ...` | Delegação de código ao bloco pai | `ATTRIBUTE.PASS_CODES` |
| `while(CODIGO)` | Repetição condicional do bloco | `ATTRIBUTE.LOOP_WHILE` |
| `param=valor` | Argumento literal | `ATTRIBUTE.EVALUATION` → `EVALUATION.LITERAL` |
| `param=&var` | Argumento por referência a variável | `ATTRIBUTE.EVALUATION` → `EVALUATION.REFERENCE` |
